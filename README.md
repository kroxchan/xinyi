# 心译

> 发出去之前，先译一下。

心译不是“又一个 AI 聊天工具”。它读你的真实微信聊天记录，学会你和 TA 的说话方式，然后在你发消息之前帮你「翻译」——把冲动的话变成准确的话，把猜测变成理解。

---

## 为什么做这个

恋爱中 80% 的争吵不是因为观点不同，而是因为表达方式不对。

你明明想表达「我需要你」，说出口却变成「你从来不在意我」。TA 本来想说「我也很累」，但你听到的是「你的感受不重要」。

**心译解决的就是这个翻译问题。**

它不会替你做决定，也不会教你话术。它的方式是——用你自己的聊天数据，帮你看清三件事：

1. **你/TA的表达习惯**——生气时真正的模式是什么
2. **你/TA 的思维方式**——TA 为什么会那样说
3. **你们之间的信号**——哪些话是雷区，哪些话其实是在说「我爱你」

---

## 核心能力

### 1. 对话引擎：不是角色扮演，是真正的「学会你」

```
普通 AI：给它一段人设描述 → 它开始表演
心译：读你的 5000 条聊天记录 → 它变成你
```

- **内心独白机制**：每次回复前，引擎先做一轮「情绪评估」——对方什么情绪、我的第一反应是什么、我现在的心理状态——然后才决定怎么说。不是直接生成回复，是先想再说
- **真实对话风格采样**：从你的聊天记录向量库中，实时抽取和当前话题最相似的几段真实对话，作为语气参照。不是手写的示例，是你真正说过的话
- **双模式切换**：「个人模式」用你的语气回应，「对象模式」用 TA 的语气对话——同一句话，从两个角度看，误解就消失了

### 2. 思维画像：从 5000 条消息里提取你的操作系统

不是让你做心理测试，也不是让 AI 帮你「总结性格」。

心译的做法是：
1. 把你的聊天记录按情境分桶——甜蜜时怎么说、吵架时怎么说、脆弱时怎么说、日常闲聊怎么说
2. 每个情境下提取你的认知模式——你的决策逻辑、你的情绪触发点、你的回避模式
3. 跨情境交叉验证——只保留在多种场景下都一致的模式
4. 压缩成可执行指令——不是「他是个敏感的人」，而是「当对方超过 3 小时未回复时，倾向于主动发消息确认关系安全感」

**这就是为什么心译的分身回复不像 AI，像你。**

### 3. 信念图谱：你相信什么，你自己可能都没意识到

从聊天记录中提取的不是关键词，是**立场**：

- 「关于加班：认为偶尔可以接受，但不应该常态化（前提：有对等的回报）」
- 「关于道歉：倾向于先冷静再处理，而不是当场解决」
- 「关于金钱：回避直接讨论，但会通过行为暗示底线」

这些信念会自动参与每次对话——当话题涉及你在意的事时，分身不会随便敷衍，会用你真正的态度回应。而且系统会自动检测信念之间的矛盾，生成验证任务帮你厘清。

### 4. 情绪状态机：不是标签，是动力学

普通情感分析给你一个标签：「正面 / 负面 / 中性」。

心译模拟的是**情绪的物理规律**：

- **惯性**：刚吵完架，不可能一秒钟变开心。负面情绪的衰减速度比正面慢
- **传染**：对方的情绪会影响你的状态，但影响权重因人而异
- **阈值**：每个人有不同的「炸点」——心译从真实记录中学到你的阈值在哪里

这意味着在连续对话中，分身的情绪变化是**连贯的**，不会上一句还在生气，下一句突然温柔。

### 5. 认知校准：不问你「你觉得你是什么性格」

8 类情境任务，不是问卷：

| 维度 | 例子 |
|------|------|
| 价值权衡 | 朋友找你借一大笔钱，你怎么决定？ |
| 道德困境 | 发现同事在偷偷做对你有利但不道德的事 |
| 冲突处理 | 和父母在重大决定上意见相反 |
| 信任校准 | 对方说了一句你不确定是不是在开玩笑的话 |
| 情绪回应 | 你努力准备的东西被否定了 |
| 身份边界 | 对方要求你改变一个你很看重的习惯 |

系统从你的选择中**反推**你的决策逻辑，而不是相信你的自我描述。推断出的信念和矛盾会反馈进信念图谱，形成闭环。

### 6. 关系顾问 @KK：不是客服，是读过你们聊天记录的朋友

在对话中输入 `@KK`，关系顾问介入。和通用 AI 咨询师的区别：

- KK 读过你们的完整聊天摘要，知道你们的沟通模式
- KK 了解 TA 的思维模型，能解释「TA 为什么会那样说」
- KK 用微信短句说话，不用咨询师腔调
- 冲突调解时，KK 会从聊天记录中调取相似的历史冲突片段作为参照

## 技术架构

```
微信加密数据库
    ↓ SQLCipher 解密（内存密钥提取）
原始消息
    ↓ 清洗（去系统消息、PII 脱敏、群聊过滤）
对话片段
    ↓ 按情境分桶 + 向量化
    ├── ChromaDB 向量记忆（对话检索 + few-shot 采样）
    ├── MemoryBank 结构化事实（置信度加权）
    ├── BeliefGraph 信念图谱（立场 + 条件 + 来源）
    ├── 思维画像（IF-THEN 指令集）
    └── 情绪档案（表达习惯 + 阈值 + 触发器）
         ↓
    多层 Prompt 组装
    ├── Tier 0: 身份锚定（你是谁，你在和谁说话）
    ├── Tier 1: 思维模型 + 说话风格
    ├── Tier 2: 情绪状态 + 过渡逻辑
    └── Tier 3: 实时注入（内心独白 + 记忆 + 信念 + few-shot）
         ↓
    对话输出（微信短句风格）
```

## 快速开始

### 环境要求

- Python 3.10+
- 一个 OpenAI 兼容的 API Key（OpenAI / Claude / DeepSeek / 国内中转均可）

### 安装

```bash
git clone https://github.com/kroxchan/xinyi.git
cd xinyi
pip install -r requirements.txt
```

### 配置 API

```bash
cp config.example.yaml config.yaml
```

编辑 `config.yaml`：

```yaml
api:
  api_key: "sk-xxxx"          # 你的 API Key
  base_url: "https://..."     # 非 OpenAI 官方时填入
  model: "gpt-4o"             # 使用的模型
```

或用 `.env`（不会被 git 提交）：

```bash
echo 'WECHAT_TWIN_API_KEY=sk-xxxx' > .env
```

### 启动

```bash
# macOS / Linux
python src/app.py

# Windows（需管理员终端）
# Win+X → "PowerShell（管理员）" → cd 到项目目录
python src/app.py
```

浏览器打开 **http://localhost:7872**，跟着向导走：**连接 → 选择 TA → 学习** → 完成。

## 平台支持

| 平台 | 数据解密 | 说明 |
|------|---------|------|
| macOS | ✅ 全自动 | 首次需 ad-hoc 重签微信 |
| Windows 10/11 | ✅ 全自动 | 需管理员终端启动 |

## 隐私

- **所有数据本地存储**，不上传到任何服务器
- API 调用仅包含当前对话上下文的摘要片段，不发送完整聊天记录
- 训练数据、向量库、人格画像全部在 `data/` 目录，删掉即清空

## 常见问题

**支持哪些 API？**
OpenAI、Anthropic、DeepSeek，以及所有兼容 OpenAI 格式的中转服务。

**需要多少聊天记录？**
至少 30 条与对象的有效聊天。300 条以上效果明显提升，1000 条以上趋于稳定。

**会影响微信正常使用吗？**
不会。解密过程只读取微信进程内存中的加密密钥，不修改微信文件或行为。macOS 首次需要重签名（去掉 Hardened Runtime），向导内自动完成。

**Windows 提示 Access Denied？**
没有用管理员终端启动。Win+X → PowerShell（管理员）→ 重新运行。

**macOS 提示 task_for_pid failed？**
需要调整 SIP，向导的「连接」页有详细步骤。
